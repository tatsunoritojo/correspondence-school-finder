# 通信制高校診断サイト 改修・機能追加指示書

**作成日:** 2025/12/22
**対象:** 開発チーム
**概要:** モニターテストからのフィードバックに基づく、UI/UX修正および機能追加の実装指示。

---

## 1. 共通・全体 (Global)

### [TXT] サービス名表記の統一
- [ ] **現状:** サイト内で `One Drop`, `OneDrop` などの表記揺れがある。
- [ ] **指示:** 全サイト内の表記を **`One drop`**（dは小文字）に統一置換すること。

---

## 2. 診断・設問画面 (Questionnaire UI)

### [UX] [高] 選択時の遷移挙動の変更
- [ ] **現状:** 選択肢をタップすると即座に次の設問へ遷移してしまい、誤操作時に戻る手間が発生する。
- [ ] **指示:**
    - ラジオボタン選択時の「自動遷移」を廃止する。
    - 「次へ」ボタンを押下して初めて遷移する仕様に変更する。
    - （または、選択後にユーザーが認知できる程度のDelayとフィードバックアニメーションを実装する）

### [UI] [中] 【保護者用】Q1文言の明確化
- [ ] **現状:** Q1に「今のあなた」という文言があり、保護者モード利用時に「親自身」か「子供」か対象が不明瞭。
- [ ] **指示:** - 保護者モード選択時、対象が「お子様」であることが分かるよう改修する。
    - 実装案：ラベルまたは補足テキスト（例：「※お子様の状況についてお答えください」）を動的に表示する。

### [FEAT] [要検討] 設問追加：通学距離・エリア
- [ ] **要望:** 「自宅からの距離」を重視して学校を選びたいニーズへの対応。
- [ ] **指示:** - 診断フローに「通学距離・通学時間」に関する設問を追加する。
    - 結果ロジック（学校の所在地データとのマッチング）に反映させる。
    - *※DBへのカラム追加およびマッチングロジックの修正が必要。*

### [FEAT] [要検討] 設問追加：入試内容
- [ ] **要望:** 面接、小論文、筆記試験など、入試方式の希望を反映させたい。
- [ ] **指示:** - 診断フローに「希望する入試スタイル（面接重視、筆記も可など）」の設問を追加する。
    - 学校データの入試情報とマッチングさせる。
    - *※学校データ側に入試方式タグの追加が必要。*

---

## 3. 診断結果画面 (Result UI)

### [UI] [中] スクロール誘導の強化（スマホ版）
- [ ] **現状:** スマホ閲覧時、ファーストビューでコンテンツが完結しているように見え、下部に詳細情報があることに気づきにくい。
- [ ] **指示:** - ファーストビュー下部に「下向き矢印（Scroll down）」アイコン、またはバウンドアニメーション等を追加する。
    - CSS調整により、下のコンテンツを少し見切れさせる（チラ見せする）ことでスクロールを促す。
