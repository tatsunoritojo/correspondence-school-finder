# QAテスト計画: データ収集フォーム

## テスト対象
- 診断サイト: https://shindan.kodomo-shinro.jp/
- データ収集フォーム（ResultPage 下部）
- Netlify Function: collect-data
- Google Sheets 書き込み

## 前提条件
- 診断を最後まで完了し、結果ページに到達すること
- テスト前に localStorage をクリアすること（DevTools → Application → Local Storage → Clear）

---

## テストケース一覧

### A. 基本フロー

| ID | テスト名 | 手順 | 期待結果 |
|----|---------|------|----------|
| A-1 | 初回送信（全項目入力） | localStorage クリア → 診断完了 → 結果ページ下部へスクロール → 「協力する」→ 全項目入力 → 「送信する」 | thanksメッセージ表示。スプレッドシートに全項目が正しく記録。revision=0 |
| A-2 | 初回送信（最小入力） | localStorage クリア → 「協力する」→ 何も入力せず「送信する」 | thanksメッセージ表示。スプレッドシートに空欄ありで記録。revision=0 |
| A-3 | 見送り | localStorage クリア → 「今回は見送る」 | 「データ収集を見送る設定になっています」+「やはり協力する」表示 |
| A-4 | 見送り後に協力 | A-3の後 → 「やはり協力する」 | askステップ（協力する/見送る）が再表示 |
| A-5 | フォームの「やめる」 | 「協力する」→ フォーム表示 → 「やめる」 | 見送り状態になる |

### B. 修正フロー

| ID | テスト名 | 手順 | 期待結果 |
|----|---------|------|----------|
| B-1 | 回答修正 | A-1完了後 → 「閉じる」→「回答を修正・追加する」→ 内容変更 → 「送信する」 | スプレッドシートに新行追加。同じsubmissionId、revision=1 |
| B-2 | 修正時のUI確認 | B-1のフォーム表示時 | askステップをスキップしフォーム直接表示。「最新の内容で上書き保存されます」テキストあり |
| B-3 | 複数回修正 | B-1完了後 → 再度修正 | revision=2 で新行追加 |

### C. エラー・エッジケース

| ID | テスト名 | 手順 | 期待結果 |
|----|---------|------|----------|
| C-1 | 通信エラー | 機内モード or DevTools で Network offline → 送信 | エラー画面表示。「フォームに戻る」で入力内容が保持されている。localStorageは未更新 |
| C-2 | エラー後の再送 | C-1の後 → ネットワーク復旧 → 「フォームに戻る」→「送信する」 | 正常に送信成功。thanks表示 |
| C-3 | 二重送信防止 | 送信ボタンを素早く連打 | 1回のみ送信。ボタンが無効化され「送信中...」表示。スプレッドシートに1行のみ |
| C-4 | 自由記述500文字制限 | 500文字以上入力を試みる | 500文字で入力が止まる。カウンター表示「500/500」 |
| C-5 | localStorage クリア後の再訪 | 送信済み → localStorage クリア → ページ再読込 | 初回フローが表示される（新しいsubmissionIdが発行される） |

### D. データ整合性

| ID | テスト名 | 確認内容 |
|----|---------|----------|
| D-1 | 日本語データ | 都道府県・自由記述が文字化けなく記録されている |
| D-2 | submissionId | 同一ユーザーの初回と修正で同じ値 |
| D-3 | revision | 初回=0、修正1回目=1、修正2回目=2 |
| D-4 | formVersion | 全行で 1 |
| D-5 | diagnosisVersion | 全行で 1 |
| D-6 | スコア値 | AX01〜AX08 に数値が入っている |
| D-7 | timestamp | ISO 8601形式 |

### E. 表示・レスポンシブ

| ID | テスト名 | 手順 | 期待結果 |
|----|---------|------|----------|
| E-1 | モバイル表示 | スマホ（375px）で結果ページ | フォームが画面幅に収まり、全項目が操作可能 |
| E-2 | デスクトップ表示 | PC（1280px）で結果ページ | フォームが適切な幅で中央寄せ |
| E-3 | フェードインアニメーション | 結果ページをスクロールしてフォームに到達 | オレンジボーダー付きでフェードイン表示 |
| E-4 | 協力済み表示 | 送信完了後にページ再読込 | 「データ収集にご協力いただきました」+「回答を修正・追加する」 |
| E-5 | 見送り表示 | 見送り後にページ再読込 | 「見送る設定になっています」+「やはり協力する」 |

### F. 満足度★タップ

| ID | テスト名 | 手順 | 期待結果 |
|----|---------|------|----------|
| F-1 | ★選択 | ★3をタップ | ★1〜3がオレンジ、★4〜5がグレー |
| F-2 | ★変更 | ★3選択後に★5をタップ | ★1〜5が全てオレンジ |
| F-3 | ★未選択で送信 | ★を選ばずに送信 | 正常送信。satisfaction は空欄 |
